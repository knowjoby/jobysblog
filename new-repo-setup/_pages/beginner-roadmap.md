*For PMs new to AI — typically 0–12 months of AI product experience.*

---

## Stage 1: Build Your AI Foundation (Weeks 1–4)

**Goal:** Understand enough AI/ML to have credible conversations with engineers.

- Learn what ML models actually do (classification, regression, recommendation, generation)
- Understand the difference between rule-based systems and ML systems
- Get comfortable with basic data concepts: training data, features, labels, model accuracy
- Read: *AI for Everyone* (Andrew Ng, free on Coursera)

---

## Stage 2: Learn the AI Product Lifecycle (Weeks 5–8)

**Goal:** Know how AI products are built, from data to deployment.

- Data collection → model training → evaluation → deployment → monitoring
- Understand why AI products fail (bad data, distribution shift, edge cases)
- Learn about model versioning and retraining schedules
- Shadow a data scientist or ML engineer for one sprint

---

## Stage 3: Ship Your First AI Feature (Weeks 9–16)

**Goal:** Own an AI feature end-to-end.

- Write an AI product spec (include data requirements, fallback behavior, confidence thresholds)
- Define success metrics: precision vs. recall trade-offs for your use case
- Plan for the human-in-the-loop: when does the AI defer to a human?
- Launch with a rollout strategy (shadow mode → 1% → full launch)

---

## Stage 4: Measure and Iterate (Ongoing)

**Goal:** Build a feedback loop between user behavior and model improvement.

- Track both product metrics (engagement, retention) and model metrics (accuracy, latency)
- Set up a process for collecting edge case feedback from users
- Run your first model retrain based on new production data

---

## Key Tools to Know
- **Notebooks:** Jupyter (to read, not write code)
- **Tracking:** MLflow, Weights & Biases (concepts)
- **Deployment:** basic understanding of APIs and inference endpoints
